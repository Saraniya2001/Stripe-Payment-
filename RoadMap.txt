🟣 PHASE 4 — Create Subscription API
🪜 Backend Endpoint
👉 Create:
POST /api/subscription/create
🧠 What it does:
Get logged-in user
Create Stripe customer (if not exists)
Create checkout session
Return session URL
💡 Flow
Frontend → request subscription
Backend → Stripe checkout session
Frontend → redirect to Stripe


🟣 PHASE 5 — Stripe Checkout
🪜 Backend Code (Concept)
const session = await stripe.checkout.sessions.create({
  payment_method_types: ["card"],
  mode: "subscription",
  customer: customerId,
  line_items: [
    {
      price: priceId,
      quantity: 1,
    },
  ],
  success_url: "http://localhost:3000/success",
  cancel_url: "http://localhost:3000/cancel",
});


🟣 PHASE 6 — Webhook (VERY IMPORTANT 🔥)

👉 This is the heart of subscription system

🧠 Why Webhook?

Stripe → tells your backend:

Payment success ✅
Subscription created ✅
Subscription canceled ❌
🪜 Steps:
Create endpoint:
POST /api/webhook
Verify Stripe signature
Handle events:
🎯 Required Events
checkout.session.completed
invoice.payment_succeeded
customer.subscription.deleted
🧠 What You Do Inside
On success:
Update user:
- subscription = starter/pro
- status = active
On cancel:
Update:
- status = canceled
